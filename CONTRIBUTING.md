# Contributing to the Agentic Engineering Handbook

Thank you for your interest in contributing! This handbook is a community-driven resource, and we welcome contributions of all kinds.

## How to Contribute

### Submitting Content

1. **Fork** the repository and clone it locally.
2. **Create a branch** using the naming convention: `docs/<topic>` (e.g., `docs/prompt-chaining`).
3. **Write your content** in the appropriate section under `docs/`.
4. **Submit a Pull Request** against `main`.

### Running Locally

```bash
# Clone the repo
git clone https://github.com/nhkhang/agentic-engineering-handbook.git
cd agentic-engineering-handbook

# Install dependencies
npm install

# Start the dev server
npm start
```

The site will be available at `http://localhost:3000`.

### PR Workflow

1. Ensure your branch is up to date with `main`.
2. One topic per PR — keep changes focused.
3. Fill out the PR template completely.
4. Address any review feedback promptly.

### Branch Naming Convention

Use the format `docs/<topic>`, for example:

- `docs/prompt-chaining`
- `docs/tool-use-patterns`
- `docs/eval-frameworks`

### Writing Style

- **Concise**: Get to the point. Avoid filler.
- **Practical**: Include examples, code snippets, or diagrams where helpful.
- **Example-driven**: Show, don't just tell. Real-world scenarios are preferred.
- **Neutral**: Avoid promoting specific vendors unless comparing objectively.

### What Makes a Good Contribution

- Fills a gap in the existing handbook
- Is accurate and up to date
- Includes practical examples or references
- Does not contain proprietary or confidential information

## Code of Conduct

All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Questions?

Open an issue using the [Content Request](https://github.com/nhkhang/agentic-engineering-handbook/issues/new?template=content-request.md) template if you'd like to propose a new topic before writing.
